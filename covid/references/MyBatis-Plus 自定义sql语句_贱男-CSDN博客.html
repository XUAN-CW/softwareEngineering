MP自带的条件构造器虽然很强大，有时候也避免不了写稍微复杂一点业务的sql，那么那么今天说说MP怎么自定义sql语句吧。

二、配置

自定义的sql当然是写在XML文件中的啦，那么首先来定义xml文件的位置，在yml配置文件如下

  1. mybatis-plus:
  2.   # 如果是放在src/main/java目录下 classpath:/com/*/*/mapper/*Mapper.xml
  3.   # 如果是放在resource目录 classpath:/mapper/**.xml
  4.   mapper-locations: classpath:/mapper/**.xml

三、具体实现

使用注解实现：

在我们Mapper接口中定义自定义方法即可。

  1. /**
  2.  * @Auther: IT贱男
  3.  * @Date: 2019/6/10 14:40
  4.  * @Description: User对象持久层
  5.  */
  6. public interface UserMapper extends BaseMapper<User> {
  7.  
  8.     /**
  9.      *
  10.      * 如果自定义的方法还希望能够使用MP提供的Wrapper条件构造器，则需要如下写法
  11.      *
  12.      * @param userWrapper
  13.      * @return
  14.      */
  15.     @Select("SELECT * FROM user ${ew.customSqlSegment}")
  16.     List<User> selectByMyWrapper(@Param(Constants.WRAPPER) Wrapper<User> userWrapper);
  17.  
  18.     /**
  19.      * 和Mybatis使用方法一致
  20.      * @param name
  21.      * @return
  22.      */
  23.     @Select("SELECT * FROM user where name = #{name}")
  24.     List<User> selectByName(@Param("name") String name);
  25.  
  26. }

使用xml文件实现：

使用xml一定要指定xml文件所在位置

  1. /**
  2.  * @Auther: IT贱男

  4.  * @Description: User对象持久层
  5.  */
  6. public interface UserMapper extends BaseMapper<User> {
  7.  
  8.     /**
  9.      *
  10.      * 如果自定义的方法还希望能够使用MP提供的Wrapper条件构造器，则需要如下写法
  11.      *
  12.      * @param userWrapper
  13.      * @return
  14.      */
  15.     List<User> selectByMyWrapper(@Param(Constants.WRAPPER) Wrapper<User> userWrapper);
  16.  
  17.     /**
  18.      * 和Mybatis使用方法一致
  19.      * @param name
  20.      * @return
  21.      */
  22.     List<User> selectByName(@Param("name") String name);
  23.  
  24. }
  1. <?xml version="1.0" encoding="UTF-8" ?>
  2. <!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN" "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
  3. <mapper namespace="com.example.demo.mapper.UserMapper">
  4.  
  5.     <select id="selectByName" resultType="com.example.demo.model.User">
  6.         SELECT * FROM user where name = #{name}
  7.     </select>
  8.

  10.         SELECT * FROM user ${ew.customSqlSegment}
  11.     </select>
  12.  
  13. </mapper>
